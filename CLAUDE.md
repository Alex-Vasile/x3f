# CLAUDE.md — x3f_tools Codebase Guide

## Project Overview

**x3f_tools** is a C/C++ library and command-line toolkit for reading and processing Sigma X3F raw image files. It supports conversion to DNG, TIFF, and PPM formats and handles every generation of Sigma camera from SD9/SD10/SD14 through the Quattro and SD Quattro H series.

The repository contains two end-user tools:
- `x3f_extract` — primary conversion tool (DNG, TIFF, PPM, JPEG thumbnail extraction, metadata, histograms)
- `x3f_io_test` — developer/debug tool that parses and prints X3F file structure

A third binary, `x3f_matrix_test`, validates matrix computation code.

## Build System

### Prerequisites

- `gcc` / `g++` (native) or the appropriate MinGW/apple-darwin cross-compilers
- `gmake`
- `cmake` (for building OpenCV)
- `zlib` development headers (Linux only; other platforms use OpenCV's bundled zlib)

### How Builds Work

1. `sys.mk` is included by both `makefile` and `src/makefile`. It detects `HOST` (e.g., `linux-x86_64`) and, unless `TARGET` is set, uses `HOST` as the default target.
2. The top-level `makefile` ensures OpenCV is built first (`install_opencv.sh`), then delegates to `src/makefile`.
3. `src/makefile` compiles all `.c`/`.cpp` sources with `-O3 -Wall`, writes `.o` and `.d` files to `../bin/<TARGET>/`, and links the three executables.

### Common Make Targets

| Command | Description |
|---|---|
| `make` | Build all three executables for the native target |
| `make dist` | Build + create distribution archive (`.tar.gz` on Linux/macOS, `.zip` on Windows) |
| `make clean` | Remove `.o`, `.d`, and intermediate dist directories |
| `make clobber` | `clean` plus removes binaries and archives |
| `make check` | Run the full BDD test suite (requires test files, see Testing) |
| `make dist-all` | (Linux host only) Build for Linux x86_64, Windows 32/64, and macOS universal |
| `make clean_opencv` | Remove all built OpenCV artifacts in `deps/` |

### Cross-Compilation Targets

Set `TARGET` on the command line:

| TARGET | Compiler used |
|---|---|
| `linux-x86_64` | `gcc` / `g++` (native) |
| `windows-x86_64` | `x86_64-w64-mingw32-gcc/g++` |
| `windows-i686` | `i686-w64-mingw32-gcc/g++` |
| `osx-x86_64` | `x86_64-apple-darwin11-gcc/g++` |
| `osx-i386` | `i386-apple-darwin11-gcc/g++` |
| `osx-universal` | Both osx-x86_64 and osx-i386 combined with `lipo` |

Example: `make TARGET=windows-x86_64 dist`

### Versioning

Version strings are produced by `git describe --always --dirty --tags` and embedded at compile time via a dedicated object file:
```
$(BINDIR)/x3f_version-$(VERSION).o
```
The full version string has the form `x3f_tools-<TAG>-<TARGET>`.

## Source Module Reference

All source files live in `src/`. Every module is a paired `.c`/`.h` (or `.cpp`/`.h`).

| Module | Role |
|---|---|
| `x3f_io.c/.h` | Core X3F file parsing: reads headers, directories, PROP/IMAG/CAMF sections. Supports format versions 2.0–4.1. |
| `x3f_process.c/.h` | High-level image processing pipeline (demosaic, white balance, colour conversion) |
| `x3f_extract.c` | CLI entry point; parses flags and drives the pipeline |
| `x3f_io_test.c` | Debug/dev entry point; prints parsed structure |
| `x3f_matrix_test.c` | Unit-tests matrix operations |
| `x3f_output_dng.c/.h` | Writes Adobe DNG files with embedded colour conversion data |
| `x3f_output_tiff.c/.h` | Writes 16-bit TIFF files |
| `x3f_output_ppm.c/.h` | Writes PPM (binary or ASCII) files |
| `x3f_image.c/.h` | Image data structures and pixel operations |
| `x3f_meta.c/.h` | CAMF/metadata extraction |
| `x3f_print_meta.c/.h` | Human-readable metadata formatting |
| `x3f_histogram.c/.h` | Histogram (linear and log-scale) CSV output |
| `x3f_denoise.cpp/.h` | Standard denoising algorithm |
| `x3f_denoise_aniso.cpp/.h` | Anisotropic (edge-preserving) denoising |
| `x3f_denoise_utils.cpp/.h` | Shared denoising helpers |
| `x3f_spatial_gain.c/.h` | Spatial gain (per-pixel colour compensation) |
| `x3f_matrix.c/.h` | General-purpose matrix math used for colour transforms |
| `x3f_dngtags.c/.h` | DNG metadata tag definitions and helpers |
| `x3f_dump.c/.h` | Binary data dump utilities |
| `x3f_printf.c/.h` | Custom formatted output (wraps fprintf with log levels) |
| `x3f_version.c/.h` | Exposes the compile-time version string |
| `mingw_dowildcard.c` | Windows wildcard expansion (linked only on Windows targets) |

## Coding Conventions

### Language and Standards

- Core library: **C** (no C99/C11 dialect flag is set explicitly, so the compiler default applies)
- Denoising modules: **C++** (required for OpenCV APIs)
- All public headers use `extern "C" { }` guards so C++ callers can link against C modules cleanly

### Naming

| Entity | Convention | Example |
|---|---|---|
| Functions / variables | `x3f_` prefix, snake_case | `x3f_load_image_block()` |
| Types (typedef) | `x3f_` prefix, `_t` suffix | `x3f_denoise_type_t` |
| Macros | ALL_CAPS | `GET1`, `FREE`, `X3F_VERSION` |
| Constants / identifiers | ALL_CAPS with `X3F_` prefix | `X3F_FOVb`, `X3F_PROP` |
| Header guards | `X3F_<MODULE>_H` | `#ifndef X3F_IO_H` |

### Code Style

- **Indentation:** spaces (2-space in macro bodies; larger in regular C code — follow the surrounding context)
- **Braces:** K&R style
- **Macros:** wrap all parameters in parentheses
- **I/O macros:** `GET1`/`GET2`/`GET4`/`GETN` are used for all binary file reads; prefer these over raw `fread` calls
- **Memory management:** use the `FREE(ptr)` macro (sets pointer to NULL after freeing) rather than bare `free()`
- **Error handling:** fatal I/O errors call `EXIT`; non-fatal conditions log via `x3f_printf`

### File Format Notes

- X3F files are **little-endian**; multi-byte integers are converted to native endian on load
- Byte streams remain in little-endian order; multi-byte element streams store each element in native endian

## Key Algorithms and Concepts

### Image Processing Pipeline (`x3f_process.c`)

1. Load raw sensor data via `x3f_io`
2. Apply spatial gain compensation (`x3f_spatial_gain`) — disabled for Quattro series
3. Bad pixel correction
4. Demosaic / unpack the Foveon X3 three-layer data
5. White balance and colour matrix transformation (`x3f_matrix`)
6. Denoising (`x3f_denoise` / `x3f_denoise_aniso`) — enabled by default since v0.54
7. Crop to active image area — enabled by default since v0.54
8. Output to chosen format

### Supported X3F Format Versions

| Version constant | Cameras |
|---|---|
| 2.0, 2.1 | SD9, SD10 |
| 2.2 | SD14 |
| 2.3 | DP1/DP2/DP3 (TRUE I/II engines) |
| 3.0 | SD1 / Merrill series |
| 4.0 | Quattro series |
| 4.1 | SD Quattro H |

### Output Formats

| Flag | Output | Notes |
|---|---|---|
| *(default)* / `-dng` | `.dng` | Linear raw + colour conversion metadata; use in Lightroom/ACR |
| `-tiff` | `.tif` | 16-bit, fully converted; defaults to sRGB since v0.54 |
| `-ppm` | `.ppm` | Binary PPM |
| `-ppm-ascii` | `.ppm` | ASCII PPM |
| `-jpg` | `.jpg` | Extracts embedded JPEG thumbnail |
| `-raw` | `.raw` | Undecoded raw sensor data |
| `-meta` | `.meta` | Metadata dump |
| `-histogram` | `.csv` | Linear histogram |
| `-loghist` | `.csv` | Logarithmic histogram |

### Processing Switches

```
-color <sRGB|AdobeRGB|ProPhotoRGB>   colour space for TIFF/DNG
-unprocessed                          raw linear data without processing
-qtop                                 Quattro top-layer only
-no-crop                              skip active-area cropping
-no-denoise                           skip denoising
-no-sgain                             skip spatial gain
-no-fix-bad                           skip bad pixel correction
-wb <WB>                              white balance preset
-compress                             ZIP compression (DNG/TIFF)
-ocl                                  enable OpenCL GPU acceleration
```

## Testing

### Framework

Tests use **Behave** (Python BDD / Gherkin) with MD5-based regression assertions. Every change that affects pixel output will require updating expected MD5 hashes in `features/consistency.feature`.

### Setup

```bash
# One-time: clone test files (~90 MB, pinned commit)
# The makefile does this automatically via the 'test_files' target

# One-time: install Python virtualenv with behave
virtualenv venv
venv/bin/pip install -r requirements.txt
```

### Running Tests

```bash
# Full test cycle (builds dist binary, clones test files, runs behave)
make check
```

The `check` target:
1. Builds the `dist` binary for the current target
2. Clones/updates `x3f_test_files` to the pinned commit (`X3F_TEST_FILES_COMMIT` in `makefile`)
3. Runs `behave` with `DIST_LOC` pointing at the built `x3f_extract` binary

### Test Scenarios

Scenarios in `features/consistency.feature` cover:

- Basic format conversions (DNG, TIFF, PPM, JPG, META, RAW, HISTOGRAM, LOGHIST)
- Compressed output (ZIP-compressed DNG and TIFF)
- Denoised conversions to DNG
- Denoised cropped colour TIFF conversions
- TIFF output variants: CROP, UNPROCESSED, QTOP, COLOR_SRGB, COLOR_ADOBE_RGB, COLOR_PROPHOTO_RGB

### Updating Expected Hashes

When a deliberate algorithm change alters output:
1. Run `x3f_extract` manually with the new binary on each test image
2. Compute MD5: `md5sum <output_file>`
3. Update the corresponding `md5` column in `features/consistency.feature`

## Adding Support for a New Camera Model

1. Identify the X3F format version the camera uses (check `x3f_io.h` version constants).
2. If it is a new version, add the version constant to `x3f_io.h` and handle it in `x3f_io.c`.
3. Obtain and embed the correct colour matrices in `x3f_process.c` / `x3f_matrix.c`.
4. Verify spatial gain behaviour (Quattro-class cameras have it disabled).
5. Add test images to a fork of `x3f_test_files` and update the pinned commit + MD5 hashes in `features/consistency.feature`.
6. Document the new camera in `doc/readme.txt` and `doc/changes.txt`.

## Dependency Details

### OpenCV (built locally)

OpenCV is built as a **static library** by `install_opencv.sh` using CMake. Only three modules are linked:
- `libopencv_photo.a` — denoising
- `libopencv_imgproc.a` — image processing
- `libopencv_core.a` — core data structures

Third-party libs bundled with OpenCV and also statically linked:
- `liblibtiff.a` — TIFF I/O
- `libtbb.a` — Intel TBB threading
- `libzlib.a` — compression (not used on Linux; system `zlib` is used instead)

### Platform-Specific Link Flags

| Platform | Extra flags |
|---|---|
| Linux | `-lz -ldl -lpthread` |
| Windows | `-static` (fully static binary via MinGW) |
| macOS | `-liconv -framework OpenCL -Wl,-dead_strip_dylibs` |

## File Patterns to Ignore

The `.gitignore` excludes build artifacts. Do not commit:
- `bin/` — compiled object files and binaries
- `dist/` — distribution archives
- `deps/` — downloaded/built OpenCV sources and libraries
- `x3f_test_files/` — external test image repository
- `venv/` — Python virtualenv
- `*~` — editor backup files
